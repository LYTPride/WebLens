# 资源列表：筛选、排序与实时更新

本文说明 WebLens 中 **Pods**、**Deployments**、**StatefulSets** 等资源表格的共用列表能力（其他资源类型后续可按同一套能力扩展）。

## 关键字筛选

- 顶部 **按 Name 关键字过滤** 仅影响当前表格展示，不改变已应用的集群与命名空间范围。

## 排序

- 部分表头右侧有 **▲ / ▼**，分别表示按该列 **升序 / 降序**；同一时间只有 **一列** 排序生效。
- 未点击排序时，列表顺序与数据到达顺序一致（不经额外排序）。
- **切换左侧资源页**（如 Pods ⇄ Deployments）时，**各自记住** 上次的排序方式。
- 点击 **「刷新列表」** 会 **清空当前页的排序**，并重新拉取该页数据。

## 与 Watch 的关系

- 列表在首次加载或「刷新列表」后，会由 **Watch** 持续推送增量更新（新增/修改/删除），无需依赖高频轮询即可接近准实时。
- 若你已开启排序，界面会 **按当前排序规则持续重排**，以便反映最新状态。
- 在 **Deployments** 等页面时，后台仍会保持与当前作用域相关的 **Pods** 数据更新（便于 rollout 后切回 Pods 不滞后）；具体策略见开发文档 [资源列表数据流](../dev/resource-list-dataflow.md)。

## Age（存活时间）列

- **Age** 由资源的 `metadata.creationTimestamp` 与 **服务端时间** 共同计算：后端在 list 与 watch 中会附带 **`serverTimeMs`**，界面按该时间基准展示，而不是单纯使用你电脑的系统时间，以便与集群、`kubectl get` 的观感更一致。
- 若你 **人为修改了本机时间** 或服务端与浏览器时间差较大，列表顶部可能出现轻量提示：*本地时间与集群时间存在偏差，Age 已按服务端时间校准*；此时 Age 仍以服务端基准为准。
- 极端时钟异常下 Age 不会用 `"-"` 表示「负时长」，而是 **从 0s 起显示** ，避免误以为是数据缺失。

## 排序位置变化提示

- **仅当当前已开启排序时**：若某行因数据更新导致其在 **排序结果中的行序** 发生变化，该行会出现 **短暂的浅色高亮**（约 1.5 秒内淡出）。
- 若数据有变化但 **排序位置未变**，不会出现该提示。
- 未开启排序时，不会有此类提示。
